# 课程表 App 开发计划书

## 1. 项目目标

开发一款基于 Flutter 的手机课程表 App，支持从 `PDF` 和 `Excel` 文件中导入课表，自动识别课程信息，生成可编辑、可保存、可查看的个人课程表。

产品原则保持不变：

- 先闭环，后智能
- 先 Excel，后 PDF
- 先确认页兜底，再继续追求更高识别率

## 2. 已完成闭环

当前项目已经完成 MVP 主链路：

`导入文件 -> 自动识别 -> 结果确认 -> 保存课程表 -> 首页/课表页查看`

已经落地的核心能力：

- 导入 `Excel`
- 导入文本型 `PDF`
- 真实解析并生成 `ImportDraft`
- 确认页手动新增、编辑、删除课程
- 本地保存课程表
- 首页展示当前学期和当前周课程
- 课表页展示周视图 / 日视图
- 课程冲突提示
- 学期设置与多学期切换
- 最近导入记录展示

## 3. 当前产品能力

### 3.1 首页

- 展示当前学期课程概览
- 快速导入 PDF / Excel
- 查看最近导入记录
- 跳转设置和完整课表

### 3.2 导入流程

- 导入入口页
- 文件选择
- 解析中页面
- 确认结果页面

### 3.3 课表管理

- 保存课表到本地
- 切换当前学期
- 调整学期名称、开学日期、节次时间

### 3.4 课表查看

- 周视图
- 日视图
- 课程详情弹层
- 冲突提示

## 4. 当前限制

当前版本仍然有这些边界：

- `Excel` 解析优先支持规整课表模板
- `PDF` 解析仅面向文本型或结构较清晰的文件
- 扫描版 PDF / 图片课表仍未支持
- 识别准确率仍需要真实样例继续迭代
- 尚未支持提醒、系统日历导出、OCR、多端同步

## 5. 阶段状态同步

### 阶段 1：项目基础搭建

已完成：

- 项目结构拆分
- 主题与路由抽离
- 首页与课表页基础 UI

### 阶段 2：导入流程骨架

已完成：

- 文件选择接入
- Parsing / Review 页面
- 页面跳转与主流程串联

### 阶段 3：Excel 导入 MVP

已完成：

- `excel` 接入
- 规整课表解析
- 合并单元格与周数规则的基础支持
- 确认后保存到课表

### 阶段 4：PDF 导入 MVP

已完成：

- 文本型 PDF 文本提取
- 规则解析课程信息
- 节次时间提取的基础支持

### 阶段 5：数据落地与课表管理

已完成：

- 本地持久化
- 当前学期保存
- 多学期切换与删除
- 设置页同步更新学期配置

### 阶段 6：体验优化

已完成：

- 首页与课表页视觉优化
- 周视图 / 日视图
- 课程冲突提示
- 最近导入记录

### 阶段 7：增强功能

当前未完成，建议按这个优先级推进：

1. 样例驱动解析增强
2. 导出到系统日历或上课提醒
3. OCR 扫描版识别
4. 多端同步

## 6. 当前推荐下一步

当前最值得继续开发的不是再补基础页面，而是：

### 6.1 样例驱动解析增强

- 收集 2~3 份真实 `Excel` 课表
- 收集 2~3 份真实 `PDF` 课表
- 把样例沉淀为测试用例
- 继续扩充解析规则和回归测试

### 6.2 用户增强能力

在解析准确率稳定后，再选择其中一项继续：

- 导出到系统日历
- 上课提醒
- OCR 图片 / 扫描版识别

## 7. 当前结论

项目已经不再处于“阶段 2 骨架期”，而是已经完成了 MVP 闭环。后续开发应从“补页面”切换到“提高解析稳定性”和“追加增强功能”。
